ІО-24 Севастьянов Максим

Лабораторна робота №3

Java — це широко використовувана, високо продуктивна, об'єктно-орієнтована мова програмування, яка була розроблена компанією Sun Microsystems у 1995 році, а пізніше придбана Oracle. Вона є однією з найбільш популярних мов програмування завдяки своїй простоті, гнучкості та портативності. Основне кредо Java — «Write Once, Run Anywhere» (WORA), що означає, що програми, написані на Java, можуть працювати на будь-якій платформі, де є Java Virtual Machine (JVM). Історія Java Мова Java була створена Джеймсом Гослінгом та його командою як частина проекту "Green Project" у 1991 році. Її початкове призначення полягало в тому, щоб забезпечити розробку програмного забезпечення для смарт-пристроїв (телевізорів, медіаплеєрів тощо). Однак мова отримала ширше застосування в Інтернет-програмуванні, зокрема для створення веб-додатків. Перша офіційна версія Java (Java 1.0) була випущена в 1996 році. З того часу мова постійно вдосконалюється, додаються нові функціональні можливості, інструменти й технології. Основні Концепції Java Об'єктно-орієнтоване програмування (ООП): Java є строго об'єктно-орієнтованою мовою. Це означає, що основою будь-якої Java-програми є класи та об'єкти. Основні принципи ООП: Інкапсуляція: Об'єкти приховують свою внутрішню реалізацію і надають зовнішній інтерфейс для взаємодії. Наслідування: Класи можуть наслідувати властивості інших класів. Поліморфізм: Одна і та сама операція може виконуватись по-різному в залежності від типу об'єкта. Абстракція: Складність системи приховується через абстрагування деталей. Платформна незалежність: Java підтримує платформну незалежність за рахунок використання віртуальної машини Java (JVM). Код, написаний на Java, компілюється у байт-код, який може бути виконаний на будь-якій системі з JVM. Це дозволяє програмам Java працювати на різних операційних системах без необхідності змін у коді. Збирання сміття (Garbage Collection): Java автоматично управляє пам'яттю за допомогою збирача сміття. Це дозволяє уникнути ручного управління пам'яттю, як це робиться в деяких інших мовах, наприклад, C++. Збір сміття звільняє пам'ять, яка більше не використовується, що зменшує ймовірність витоків пам'яті. Багатопоточність (Multithreading): Java підтримує багатопоточність, що дозволяє виконувати кілька завдань одночасно. Це особливо корисно для розробки високопродуктивних програм, таких як сервери або додатки з великими обчислювальними навантаженнями. Безпека: Безпека — один із ключових аспектів Java. Програми Java виконуються у «пісочниці» (sandbox), що обмежує їх доступ до ресурсів системи. Крім того, Java включає механізми перевірки байт-коду та управління правами доступу, що зменшує ймовірність запуску шкідливого коду. Висока продуктивність: Хоча Java — це інтерпретована мова, сучасні JVM використовують методи Just-In-Time (JIT) компіляції, що дозволяє перетворювати байт-код в машинний код під час виконання. Це значно підвищує продуктивність Java-додатків. Багата бібліотека класів (Java API): Java має величезну стандартну бібліотеку класів (Java API), яка охоплює широкий спектр функціональних можливостей — від маніпуляцій зі строками, файлами та мережевими операціями до складніших технологій, таких як робота з базами даних, графікою, та багатопоточністю. Переваги Java Портативність: Програми на Java можуть виконуватись на різних платформах без зміни коду. Завдяки JVM, Java дозволяє легко переносити програми між операційними системами (Windows, macOS, Linux тощо). Простота: Java має простий синтаксис, що робить її легкою для вивчення. Вона значно простіша, ніж інші мови (наприклад, C++), особливо завдяки автоматичному управлінню пам'яттю. Масштабованість: Java-додатки легко масштабуються від маленьких програм до складних корпоративних систем. Мова підтримує модульність через пакети та має добре розвинену систему управління залежностями. Широке застосування: Java використовується в багатьох областях: веб-розробка (за допомогою сервлетів і JSP), розробка Android-додатків, розробка корпоративних систем (Java Enterprise Edition), хмарні рішення та великі дані (за допомогою платформ типу Apache Hadoop). Недоліки Java Швидкодія: Хоча сучасні JVM значно підвищили продуктивність Java, вона все ж може бути повільнішою порівняно з компільованими мовами, такими як C або C++. Інтерпретація байт-коду займає більше часу, ніж виконання машинного коду. Високі вимоги до ресурсів: Програми Java можуть вимагати більше пам'яті та процесорного часу через багатошаровість JVM і збирач сміття, що може бути недоліком для деяких типів додатків, особливо для мобільних або вбудованих систем. Технологічна екосистема Java Java SE (Standard Edition): Це базовий рівень Java, який включає JVM, основні класи та API. Вона використовується для розробки настільних додатків та базових сервісів. Java EE (Enterprise Edition): Java EE надає додаткові інструменти та бібліотеки для створення корпоративних додатків, таких як веб-додатки, системи управління базами даних, і масштабовані сервіси. Java EE включає технології, такі як сервлети, JSP, Enterprise JavaBeans (EJB), JPA та інші. Java ME (Micro Edition): Java ME призначена для мобільних пристроїв та вбудованих систем. Ця версія Java є легкою та оптимізованою для роботи на ресурсно-обмежених пристроях. Android: Android, найбільш популярна мобільна платформа у світі, використовує Java як основну мову для розробки додатків. Хоча Android використовує спеціальну віртуальну машину Dalvik (а тепер ART), синтаксис Java залишається основним для Android-додатків. Інструменти для розробки на Java IDE (Інтегровані середовища розробки): Eclipse: Одне з найпопулярніших середовищ для розробки на Java з великою кількістю плагінів та підтримкою інструментів для розробки на Java EE. IntelliJ IDEA: Потужна IDE з інтегрованими засобами для розробки веб, мобільних та корпоративних додатків. NetBeans: Відкрите середовище для розробки, що надає прості інструменти для роботи з Java SE та EE. Будування та управління залежностями: Maven: Інструмент для управління проєктами та залежностями, який автоматизує процес компіляції та збірки. Gradle: Популярна система автоматизації зборки, яка підтримує як Java, так і інші мови. Системи контролю версій: Java розробники часто використовують Git та інші системи контролю версій для управління змінами коду. Висновок Java — це універсальна мова програмування, яка вже понад два десятиліття залишається однією з найпопулярніших мов у світі. Вона пропонує високу продуктивність, простоту використання, потужну екосистему та підтримку багатьох типів програм. Завдяки своїй портативності, багатопоточності та широким можливостям використання, Java залишається вибором багатьох розробників для створення програмного забезпечення, яке працює на будь-якій платформі.

Spring — це потужний і гнучкий фреймворк для розробки Java-додатків, що надає комплексну інфраструктуру для створення корпоративних додатків. Основне завдання Spring Framework — спрощення розробки додатків, завдяки впровадженню принципів інверсії управління (IoC) і аспектно-орієнтованого програмування (AOP). Цей фреймворк став популярним завдяки своїй модульній архітектурі, гнучкості та масштабованості, що дозволяє використовувати лише необхідні компоненти для конкретного проекту. Ключові компоненти Spring Framework: 1. Spring Core Це основний модуль Spring, на якому базуються інші частини фреймворку. Він забезпечує функціонал інверсії управління (IoC), що дозволяє додатку динамічно впроваджувати залежності між компонентами. Завдяки контейнеру IoC, об'єкти створюються та зв'язуються автоматично. 2. Spring AOP (Аспектно-орієнтоване програмування) Аспектно-орієнтоване програмування дозволяє розділяти бізнес-логіку додатка від загальних технічних аспектів (наприклад, логування, безпека, транзакції). Це дозволяє зберегти код чистим та сконцентрованим на основних задачах. Spring AOP дозволяє додавати ці аспекти через конфігурацію, не змінюючи бізнес-логіку. 3. Spring MVC (Model-View-Controller) Цей модуль використовується для створення веб-додатків на основі шаблону MVC. Spring MVC дозволяє розробляти структуру веб-додатку, де логіка контролерів (Controller) розподіляється між уявленнями (View) та моделями (Model), що робить додатки гнучкими та легко масштабованими. 4. Spring Data Цей компонент спрощує роботу з базами даних, включаючи різні моделі доступу до даних: реляційні, нереляційні, кеші та інші сховища. Spring Data забезпечує легку інтеграцію з ORM-фреймворками, такими як Hibernate або JPA, що дозволяє взаємодіяти з базами даних через об'єкти Java, а не через SQL-запити. 5. Spring Security Цей модуль відповідає за безпеку додатків. Spring Security дозволяє конфігурувати автентифікацію (authentication) і авторизацію (authorization) в додатках, інтегрувати різні схеми безпеки, такі як JWT, OAuth2, LDAP та інші. Це надійний інструмент для забезпечення захисту корпоративних додатків. 6. Spring Boot Spring Boot — це набір інструментів для швидкого створення production-ready додатків. Він значно спрощує налаштування Spring, дозволяючи розробникам сконцентруватися на бізнес-логіці замість конфігурації. Spring Boot пропонує "out-of-the-box" рішення для розгортання, завдяки вбудованому веб-серверу Tomcat або Jetty, автоконфігурації та можливості створення мікросервісів. 7. Spring Cloud Цей модуль спеціалізується на підтримці хмарних рішень і мікросервісів. Spring Cloud надає інструменти для розподілених систем, таких як управління конфігурацією, балансування навантаження, маршрутизація, відкриття сервісів, моніторинг і безпека. 8. Spring Batch Модуль, призначений для обробки великих обсягів даних у пакетному режимі. Spring Batch дозволяє організовувати обробку даних, таких як читання, перетворення та запис в різні джерела.
